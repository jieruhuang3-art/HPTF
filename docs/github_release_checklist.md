## Data Policy

- Do not publish raw datasets.
- Do not publish processed TSV files unless dataset licenses explicitly allow redistribution.
- Keep only directory placeholders such as `.gitkeep`.
- Document all dataset download and preprocessing steps instead of committing data.

## Checkpoint Policy

- Do not commit `.pt`, `.pth`, `.bin`, `.ckpt`, or checkpoint directories.
- If releasing checkpoints later, publish them through a separate release asset or model hosting service with license notes.
- Record checkpoint provenance, dataset version, split seed, and training command.

## Reproducibility Commands

```bash
python -m compileall hptf
python -m hptf.training.pretrain --help
python -m hptf.training.finetune --help
python -m hptf.evaluation.evaluate --help
bash -n scripts/preprocess.sh
bash -n scripts/pretrain.sh
bash -n scripts/finetune_tor.sh
bash -n scripts/finetune_all.sh
bash -n scripts/evaluate.sh
bash -n scripts/run_ablation_tor.sh
```

Primary workflow:

```bash
bash scripts/preprocess.sh
bash scripts/pretrain.sh

```

## Known Limitations

- The real HPTF implementation is named `FlowSemMAEClassifier` in the source code.
- Some files under `hptf/models/` are wrappers or re-exports for paper-module navigation.
- Not every paper ablation has a direct CLI switch.
- No datasets, checkpoints, logs, or large generated results are included.
- README results are existing paper results; they are not regenerated by this cleanup step.

## Before-Push Checklist

- Confirm `git status` includes only intended source, docs, configs, scripts, and placeholder files.
- Confirm no dataset files, PCAP files, checkpoints, logs, caches, or large result arrays are staged.
- Run the reproducibility commands above.
- Regenerate `project_tree.txt`.
- Re-read `README.md`, `docs/code_mapping.md`, `docs/ablation.md`, and `migration_report.md` for unsupported-feature clarity.
- Keep missing or unavailable results as `TBD`; do not estimate or fill values.
